# Express TypeScript API

A minimal, production-ready REST API built with **Express.js** and **TypeScript**. This project demonstrates modern Node.js backend development with strict type checking, ES modules, and hot-reload development workflow.

---

## Features

- **Express 5** — Fast, unopinionated web framework
- **TypeScript 6** — Strict type safety with modern compiler options
- **ES Modules** — Native `import/export` syntax (`"type": "module"`)
- **Hot Reload** — Instant feedback during development with `ts-node-dev`
- **Source Maps** — Full debugging support with `.map` files
- **Declaration Maps** — Go-to-definition support for consumers

## Quick Start

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd express-ts-api
```

### 2. Install Dependencies

```bash
npm install
```

This installs:
- `express` — Web framework
- `@types/express` — TypeScript definitions for Express
- `@types/node` — TypeScript definitions for Node.js APIs
- `typescript` — TypeScript compiler
- `ts-node-dev` — TypeScript execution and hot-reload for development

### 3. Start Development Server

```bash
npm run dev
```

Your API will be available at `http://localhost:3000` (or the port configured in your app).

> The server automatically restarts when you save changes to any `.ts` file.

### 4. Build for Production

```bash
npm run build
```

This compiles TypeScript to JavaScript in the `dist/` directory with:
- JavaScript output (`.js`)
- Source maps (`.js.map`)
- Type declarations (`.d.ts`)
- Declaration maps (`.d.ts.map`)

### 5. Start Production Server

```bash
npm start
```

Runs the compiled application from `dist/index.js`.

## TypeScript Configuration Highlights

The `tsconfig.json` is configured for modern Node.js development:

- **`"module": "nodenext"`** — Full ES module support with Node.js resolution
- **`"target": "esnext"`** — Latest ECMAScript features
- **`"strict": true`** — Enable all strict type-checking options
- **`"noUncheckedIndexedAccess": true`** — Safer array/object access
- **`"exactOptionalPropertyTypes": true`** — Distinguish `undefined` from missing properties
- **`"isolatedModules": true`** — Required for Babel/ts-node compatibility
- **`"skipLibCheck": true`** — Faster builds by skipping `.d.ts` checking
- **`"sourceMap": true`** — Generate source maps for debugging
- **`"declaration": true`** — Generate `.d.ts` type declaration files

> **Tip:** Uncomment `rootDir` and `outDir` in `tsconfig.json` if you want to enforce a strict `src/` → `dist/` layout.

## Adding Routes

Create a new route file in `src/` and import it in `index.ts`:

```typescript
// src/routes/users.ts
import { Router } from 'express';

const router = Router();

router.get('/', (req, res) => {
  res.json({ users: [] });
});

export default router;
```

Then register it in your main app:

```typescript
// src/index.ts
import userRoutes from './routes/users.js';  // Note: .js extension required for ES modules

app.use('/users', userRoutes);
```

> **Important:** When using ES modules (`"type": "module"`), always include the `.js` extension in import paths, even for `.ts` files. TypeScript handles the resolution during compilation.

## Environment Variables

For production deployments, create a `.env` file (requires `dotenv` package):

```bash
npm install dotenv
```

```env
# .env
PORT=3000
NODE_ENV=production
```

Then load it at the top of `src/index.ts`:

```typescript
import 'dotenv/config';

const PORT = process.env.PORT || 3000;
```

> **Security:** Never commit `.env` files to Git. Add `.env` to your `.gitignore`.

## Deployment

### Deploy to Render / Railway / Heroku

1. Set the build command: `npm run build`
2. Set the start command: `npm start`
3. Ensure `NODE_ENV=production` is set

### Docker Deployment

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]
```

## Troubleshooting

### Error: "Cannot find module"

Make sure you include the `.js` extension in all ES module imports:
```typescript
// Correct
import { something } from './utils.js';

// Incorrect
import { something } from './utils';
```

### Error: "ts-node-dev command not found"

Reinstall dev dependencies:
```bash
npm install
```

### Port Already in Use

Change the port in your `src/index.ts` or set the `PORT` environment variable.
